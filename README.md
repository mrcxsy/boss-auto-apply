# Mr_Cheng - 岗位投递助手

> 🚀 BOSS 直聘自动投递 Chrome 扩展 — 多维度精准筛选 + AI 深度匹配

[![Chrome Extension](https://img.shields.io/badge/Chrome%20Extension-Manifest%20V3-blue)](https://developer.chrome.com/docs/extensions/)
[![Version](https://img.shields.io/badge/version-1.19.0-green)](https://github.com/mrcxsy/boss-auto-apply)
[![License](https://img.shields.io/badge/license-Apache%202.0-blue)](LICENSE)

一个运行在 BOSS 直聘网页端的 Chrome 扩展，帮助求职者自动筛选岗位、智能匹配职位并批量发起沟通，大幅提升求职效率。

---

## ✨ 核心功能

### 🎯 基础筛选
- **关键词匹配**：按职位关键词精准筛选，支持多关键词
- **排除关键词**：自动过滤含排除词的岗位（标题排除 + JD 排除）
- **薪资过滤**：设置最低薪资门槛，自动解析薪资范围（支持 `15-25K`、`面议` 等格式）
- **HR 活跃度**：过滤不活跃的 HR，避免无效投递

### 🤖 AI 深度匹配
- **JD 智能分析**：使用大模型（DeepSeek 等）分析职位描述与个人画像的匹配度
- **个性化打招呼**：根据 JD 自动生成定制化的开场白，提高回复率
- **多模型支持**：可配置多个 AI 模型（DeepSeek、OpenAI 等），自由切换
- **匹配日志**：记录每次 AI 匹配结果，方便复盘
- **结果缓存**：相同岗位的匹配结果自动缓存，避免重复调用 API

### 📋 工作经历配置
- **技能标签**：配置你的核心技能
- **工作经历**：录入工作背景供 AI 参考
- **求职要求**：设定期望条件，AI 综合评估匹配度

### 🏢 企业黑名单
- **手动添加**：一键拉黑不想投递的公司
- **自动跳过**：黑名单企业在扫描时自动过滤

### 📊 运行日志
- **实时统计**：已沟通数、已扫描数、跳过数
- **持久化日志**：页面刷新后日志不丢失
- **详细记录**：每次操作的时间、公司、岗位、结果、跳过原因

### 💾 配置管理
- **导入/导出**：一键备份和恢复所有配置（JSON 格式）
- **本地存储**：所有数据保存在 `chrome.storage.local`，不上传任何服务器

## 🛡️ 反检测

本扩展内置了一套完整的反检测体系，模拟真实人类行为模式，降低被平台识别的风险。

### 鼠标轨迹模拟

| 特性 | 实现细节 |
|------|----------|
| **贝塞尔曲线轨迹** | 三次贝塞尔曲线生成移动路径，控制点随机偏移 ±60px（水平）/ ±40px（垂直） |
| **Fitts' Law 定律** | 移动时间 `MT = a + b × log₂(D/W + 1)`，其中 a ∈ [60,100]ms，b ∈ [100,160]ms/bit |
| **高频微抖动** | 每步叠加高斯抖动（σ=1.2px），模拟手部生理性颤抖 |
| **动态过冲** | 基础过冲概率 5%，疲劳时最高 15%；过冲距离 8-30px，回调使用 easeOut 缓动 |
| **惯性漂移** | 80% 概率在点击前产生 2-4 次微漂移（衰减系数 0.6），模拟手指微调 |
| **犹豫行为** | 5% 概率在移动中暂停并反向移动 10-30%，模拟人类不确定时的犹豫 |
| **非对称加速** | 启动阶段略慢（t<0.2），中段匀速，接近目标明显减速（t>0.8） |

### 点击事件模拟

完整事件链：`pointerdown → mousedown → pointerup → mouseup → click`

| 属性 | 模拟方式 |
|------|----------|
| **压力值 (pressure)** | pointerdown: 0.3-0.7，pointerup: 0 |
| **倾斜角 (tiltX/Y)** | 随机 -15° ~ +15° |
| **时间戳单调性** | 自动修正同一毫秒内的重复 timeStamp，注入 0.1-0.6ms 偏移 |
| **isTrusted 绕过** | MAIN world 原型级 hook `EventTarget.prototype.addEventListener`，标记事件为可信 |
| **坐标一致性** | 根据鼠标坐标动态选择 dispatch 目标，防止 `elementFromPoint` 检测不一致 |

### 打字模拟

| 特性 | 数值 |
|------|------|
| **字符间隔** | 中文 40-120ms，英文词内 30-100ms，词间 150-400ms |
| **打字错误率** | 5% 概率打错相邻键位，自动退格修正 |
| **思考停顿** | 5% 概率（注意力低时更高）插入 300-500ms 停顿 |
| **词组节奏** | 文本自动分段，词组间额外停顿 100-300ms |
| **键盘事件完整性** | 完整 keydown/keyup 事件，包含 keyCode/charCode/which 三件套 |
| **中文输入法** | 每 2-5 个字符插入 100-300ms 停顿，模拟输入法选词 |

### 滚动模拟

支持 3 种滚动模式，根据兴趣因子自动选择：

| 模式 | 步数 | 缓动 | 步延迟 | 适用场景 |
|------|------|------|--------|----------|
| **快速扫视** | 4-8 步 | ease-in | 15-25ms | 低兴趣快速浏览 |
| **仔细阅读** | 10-18 步 | ease-out | 30-50ms | 高兴趣仔细阅读 |
| **触控板模拟** | 8-15 步 | linear | 20-35ms | 模拟触控板操作 |

- **回滚行为**：8% 概率（仔细阅读模式 12%）中途向上滚动 10-25% 再回来
- **WheelEvent 完整派发**：包含真实的 `deltaY`（80-150）、`deltaMode`、`clientX/Y`

### 疲劳模型

模拟人类随时间推移而产生的疲劳效应：

```
疲劳因子 = 1.0 + (基础疲劳 + 注意力周期波动) × 个体差异系数
基础疲劳 = min(0.4, 0.15 × log(1 + 运行分钟数 / 20))   // 对数增长
注意力周期 = sin(分钟数 % 90 / 90 × 2π) × 0.05         // 90分钟周期，±5% 波动
```

- **最大疲劳因子**：1.4（延迟增加 40%）
- **疲劳影响**：操作延迟增加、过冲概率升高、微抖动幅度增大、打字错误率上升

### 休息机制

| 类型 | 触发条件 | 持续时间 | 概率 |
|------|----------|----------|------|
| **短休息** | 沟通 > 2 次 且 距上次休息 > 60s | 20-50 秒 | 每次操作 5%（疲劳时翻倍） |
| **长休息** | 沟通 > 5 次 且 距上次休息 > 5 分钟 | 2-3 分钟 | 每次操作 2%（疲劳时翻倍） |
| **异常长停顿** | 随机触发 | 3-10 秒 | 每次操作 3% |

休息期间仍会偶尔移动鼠标（30% 概率/5秒）和切换页面焦点（10% 概率），模拟"人在但没操作"。

### 行为配置文件

每个用户生成独特的行为参数组合，增加批量检测难度：

| 参数 | 范围 | 说明 |
|------|------|------|
| speedFactor | 0.6-1.5 | 操作速度（与打字速度相关，相关系数 0.6） |
| typingFactor | 0.7-1.4 | 打字速度因子 |
| attentionFactor | 0.5-1.5 | 注意力集中度（影响停顿频率） |
| browsingFactor | 0.5-1.5 | 浏览习惯（快速扫视 vs 仔细阅读） |
| mousePrecision | 0.6-1.4 | 鼠标精确度（与速度负相关） |
| typoRate | 0.02-0.12 | 打字错误率基数（与精确度负相关） |

参数使用正态分布生成，能力之间存在合理相关性（如速度快的人打字也快，但鼠标精确度略低）。

### 其他防御

- **rAF 防御循环**：持续运行 `requestAnimationFrame`，每 30-80 帧执行一次轻量 DOM 读取（`offsetHeight`/`scrollTop`），模拟真实渲染循环
- **空闲鼠标移动**：每 3-8 秒随机移动鼠标（30% 概率），模拟用户浏览页面
- **焦点变化模拟**：每 10-30 秒有 10% 概率模拟页面切走/切回（blur + visibilitychange）
- **MutationObserver 弹窗检测**：实时监听 DOM 变化，检测系统弹窗（匹配 16 种选择器 + 24 个关键词）
- **非枚举全局变量**：所有模块通过 `Object.defineProperty` 挂载，`enumerable: false`，避免被 `Object.keys(window)` 探测
- **Symbol 键隐藏**：`BossUtils` 使用 `Symbol.for('__boss_utils__')` 作为键，`Object.getOwnPropertyNames` 无法发现

## 🚀 使用指南

### 基本使用流程

```
1. 打开 BOSS 直聘找工作页面
2. 点击右侧浮动设置面板
3. 配置筛选条件（城市、关键词、薪资等）
4. （可选）配置 AI 深度匹配
5. 点击「开始」按钮
6. 扩展自动扫描、筛选、发起沟通
```

### AI 深度匹配配置

1. 切换到「AI 深度匹配」标签页
2. 填入你的 AI API Key（如 DeepSeek）
3. 在「工作经历」标签页配置个人信息
4. 开启 AI 深度匹配开关
5. 扩展将自动分析 JD 并生成个性化打招呼语

### 配置导入/导出

- **导出**：AI 深度匹配标签页 → 点击「导出配置」
- **导入**：粘贴配置 JSON → 点击「导入配置」

## 🔧 技术架构

### 双世界注入

```
┌─────────────────────────────────────────────────┐
│              Chrome Extension (MV3)              │
├──────────────────────┬──────────────────────────┤
│   MAIN world         │   ISOLATED world         │
│   (document_start)   │   (document_end)         │
│                      │                          │
│   event-trusted.js   │   utils.js               │
│   ↓                  │   ↓                      │
│   Hook               │   selectors.js           │
│   addEventListener   │   ↓                      │
│   原型链             │   config.js              │
│   ↓                  │   ↓                      │
│   标记 isTrusted     │   filters.js → ai.js     │
│   = true             │   ↓                      │
│                      │   ui.js                  │
│                      │   ↓                      │
│                      │   anti-detection.js      │
│                      │   ↓                      │
│                      │   automation.js          │
│                      │   ↓                      │
│                      │   content.js (入口)      │
├──────────────────────┴──────────────────────────┤
│              background.js (Service Worker)      │
│              AI API 跨域代理                      │
└─────────────────────────────────────────────────┘
```

### PUA 字体解码

BOSS 直聘使用自定义 PUA Unicode 字体混淆薪资数字。解码流程：

1. 创建离屏 Canvas（40×40px）
2. 将 PUA 字符和标准数字 0-9 分别渲染到 Canvas
3. 逐像素比对 RGB 值，计算相似度
4. 缓存映射结果（每种字体只需解码一次）
5. Canvas 用完立即清理，不留下痕迹

### 延迟生成模型

所有人类行为延迟使用混合分布生成，而非简单的随机数：

```
70% 正态分布 — 正常操作（主要行为）
15% 对数正态分布 — 偶尔的长尾延迟（模拟分心/思考）
10% 均匀分布 — 快速连贯操作（打字连击等）
 5% 极端值 — 罕见的长时间停顿（模拟看手机/喝水）
```

### AbortController 即时停止

所有异步操作（`humanSleep`、`simulateScroll` 等）通过 `Promise.race` 与 AbortSignal 竞争，用户点击停止后所有操作立即返回，无需等待当前延迟结束。
